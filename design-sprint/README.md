# Design Sprint Workspace

This directory is the working space for your Google Design Sprint facilitated by the **DesignSprintAgent**.

## How to Use This Workspace

1. Open GitHub Copilot Chat and invoke the agent with `@DesignSprintAgent`.
2. Tell the agent which phase you are starting (or ask it to assess where you are).
3. The agent will guide you through each activity and save all outputs into this directory.

## Workspace Structure

```
design-sprint/
  sprint-tracker.md                           ← Live tracker (updated after every activity)
  phase-1-understand/                         ← Understand phase artefacts
    interview-[subject-slug]-summary.md       ← Generated per interview
    user-journey-map.md
    hmw-notes.md
  phase-2-define/                             ← Define phase artefacts
    hmw-clusters.md
    problem-statement.md
    sprint-questions.md
    user-persona.md
  phase-3-sketch/                             ← Sketch phase artefacts
    lightning-demos.md
    solution-sketch-[author-slug].md
  phase-4-decide/                             ← Decide phase artefacts
    heatmap-votes.md
    storyboard.md
```

## Design Sprint Phases

### Phase 1 — Understand 🔍
**Goal:** Build shared knowledge of the problem space.
Key activities: Sprint Brief, Expert/User Interviews, User Journey Map, "How Might We" notes.

### Phase 2 — Define 🎯
**Goal:** Focus on the highest-impact problem to solve.
Key activities: Cluster HMW notes, write Problem Statement, identify Sprint Questions, define User Persona.

### Phase 3 — Sketch ✏️
**Goal:** Generate a wide and diverse set of solutions.
Key activities: Lightning Demos, Crazy 8s, Solution Sketching, Silent Review.

### Phase 4 — Decide ✅
**Goal:** Select the best solution to move forward.
Key activities: Gallery Walk, Heatmap Voting, Speed Critique, Supervote, Storyboarding.
